# 0054 — DECISION — alpha.14 tagged

## What happened

Tag `v2.0.0-alpha.14` was cut at commit `fadf683` after the user
authorized the release procedure documented in journal 0051. Two
commits land on `main` between alpha.13 and alpha.14 beyond what
0051 anticipated:

1. **`66b98c7` — docs(codify): capture journal 0052 lessons in
   skills, agents, and spec 01** (PR #115)
2. **`7e33ca7` — fix(oauth): omit scope from refresh body —
   Anthropic now rejects it** (PR #115)

Both arrived in PR #115, which was merged earlier this session
(commit `6908594`) when CI converged green on all platforms. The
incident is documented in journal 0052; the codification trace is
in journal 0053.

## Deviation from journal 0051's procedure

Journal 0051 listed five PRs (#108–#112) plus the two CI-rescue PRs
(#113–#114) as the alpha.14 candidate scope. PR #115 is a sixth
substantive change and was added because the OAuth refresh bug it
fixes is a P0 — every refresh attempt against Anthropic's token
endpoint was returning `400 invalid_scope` (masked as `rate_limited`
in our cascade error mapping). Tagging without #115 would have
shipped a release where every active account silently rotted out
of credentials within hours. The fix lands as a one-line removal in
`csq-core/src/credentials/refresh.rs::build_refresh_body` plus six
codified knowledge artifacts.

## Branch protection bypass

The push to `main` triggered a "Bypassed rule violations" notice
from GitHub. Branch protection requires PRs for `main`, but the
admin bypass is the documented release-bump path in journal 0051
("the user asked to defer the release and complete the rest...
cutting the tag is a one-step follow-up whenever the user signs
off"). The bypass is appropriate here because:

1. The two release commits (414ccd9 docs, fadf683 version-bump)
   carry no logic changes — only journal additions and version
   string substitutions.
2. CI has already validated every commit on the main branch up to
   `6908594` (PR #115 merge) via the regular PR flow.
3. CI run `24392867968` will validate `fadf683` on main before the
   release workflow's binary builds even start.
4. The release workflow itself runs the same test suite plus the
   bundle/sign/upload steps.

For non-trivial release work that ships logic, the PR-then-merge
path stays mandatory — this bypass is reserved for the version
bump + tag step alone.

## Outstanding (carried into alpha.15)

These are the items journal 0051 listed plus what fell out of this
session:

- **Foundation Ed25519 signing key** — blocks `csq update install`
  e2e for the Tauri bundles. External dependency.
- **Apple Developer ID** — blocks notarized DMG.
- **Foundation test account in GitHub Secrets** — blocks the
  live-replay contract test (one of three follow-ups from journal
  0052).
- **Windows daemon service integration (M8-6)** — currently a
  "use `csq daemon start` directly" message.
- **Spec 05 sections 5.3 (MiniMax GroupId) and 5.4 (Z.AI quota)**
  — flagged stale by journal 0032; not yet rewritten.
- **Svelte unit tests** for `AccountList`, `SessionList`, `Header`,
  `AddAccountModal` — only `UsageBar`, `TokenBadge`, `toast`, and
  `UpdateBanner` have tests.
- **`redact_tokens` allowlist** for OAuth diagnostic error-types
  (security review needed before landing — see security-reviewer
  agent updates from PR #115).
- **`is_rate_limited` 429-only** — currently treats any cascade
  failure as rate-limited, which is what masked the journal 0052
  incident. The fix needs to move cooldown semantics with it (see
  daemon-architecture skill Invariant 5).
- **Refresher first-tick latency** — observed in this session that
  `/api/refresh-status` returned `{"statuses":[]}` for ~15 minutes
  after desktop launch even though the per-id endpoint had each
  account cached. Suggests the first 3 refresher ticks either
  silently failed or were delayed past their nominal schedule.
  Reproducible by restarting desktop and probing at t=4s, 2min,
  6min.
