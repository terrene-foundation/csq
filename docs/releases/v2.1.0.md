# csq v2.1.0

**Release date:** 2026-04-23
**Prior release:** v2.0.1 (2026-04-22)
**Checksum / signature:** see artifact `SHA256SUMS` and `.sig` files on the GitHub Releases page.

---

## What's new

v2.1.0 adds **Codex** as a first-class second surface alongside ClaudeCode. csq now manages Codex accounts end-to-end: device-auth login, central token refresh, usage polling against the live `wham/usage` endpoint, in-flight `csq swap` between Codex slots without dropping the running session, cross-surface swap between Codex and ClaudeCode with confirm-prompt + clean handover, and a desktop UI that mirrors the ClaudeCode flow with a Terms-of-Service disclosure for first-time Codex sign-in.

The release also flips the quota schema writer from v1 to v2 (the v2.0.1 dual-read landed in advance specifically for this), and ships two rounds of red-team convergence on the new Codex surface.

If you do not use Codex, you can update with no behaviour change on the ClaudeCode side. Quota.json migrates v1 → v2 silently on first daemon tick after upgrade; downgrade to v2.0.1 still reads v2 via the dual-read tolerance.

### The Codex surface

- **Surface dispatch architecture** (PR-C1, PR-C1.5). `Surface::ClaudeCode` and `Surface::Codex` enums replace the prior implicit "everything is Anthropic" assumption across `discovery`, `auto_rotate`, `rotation::swap_to`, `daemon::refresher`, and `usage_poller`. The quota schema is frozen at v2 with nested per-surface state and an `extras: Option<serde_json::Value>` escape hatch (spec 07 §7.4.1).
- **Codex login** (PR-C3, PR-C3b, PR-C3c). `csq login N --provider codex` and the desktop AddAccountModal both run the codex-cli `--device-auth` flow, parse the device-code from the scrubbed-line stdout, surface it in the UI, and stream codex-cli's progress events to the user. The login subprocess can be cancelled mid-flow via `cancel_codex_login`. Canonical credentials land at `credentials/codex-N.json` at 0o600 with an immediate flip to 0o400 by the daemon's startup reconciler (INV-P08).
- **Codex token refresh** (PR-C4). The daemon's refresher gains `broker_codex_check` + `HttpPostFnCodex` and surface-dispatches each account based on its `discovery` classification. Refresh fires 2h pre-expiry per INV-P01; clock skew is read from the response `Date` header. The Windows H2 merge gate also ships in this release: `csq-cli/src/commands/run.rs::require_daemon_healthy` is now cross-platform, and `csq-core/tests/integration_codex_refresher_windows.rs` exercises a named-pipe daemon plus surface-dispatched refresher cycle.
- **Codex usage polling** (PR-C5, **STABLE**). New `usage_poller/codex.rs` parses `wham/usage` against the schema captured live in journal 0010 (two-window rate-limit: 5h primary + 7d secondary, parallel to Anthropic; `used_percent` is 0–100, not 0–1). The §5.7 capture path is **Path A STABLE** — journal 0008 was superseded by journal 0010 once the user re-authed and the live capture completed. The poller has a 5-fail circuit breaker (15min initial backoff, cap 80min) and writes raw bodies to `accounts/codex-wham-raw.json` (0600, redactor-first) for forensic drift detection.
- **Quota schema v2 write-flip** (PR-C6). `daemon::usage_poller` now writes `schema_version = 2` to `quota.json`. The v2 shape carries nested `CounterState` / `RateLimitState` per spec 07 §7.4.1. v2.0.1's read-tolerance lands the migration cleanly: an alpha.21 / v2.0.0 / v2.0.1 quota.json round-trips through v2.1.0 without data loss. **Downgrade to v2.0.1** continues to work via the dual-read.
- **Cross-surface `csq swap`** (PR-C7). `csq swap N` resolves the target's surface from `discovery::discover_all` and routes to one of three paths:
  - **Same-surface ClaudeCode** — atomic symlink repoint (in-flight, no process restart).
  - **Same-surface Codex** — atomic symlink repoint via the new `repoint_handle_dir_codex` mirror (in-flight; no `exec`-replace, conversation preserved). M10 / journal 0023.
  - **Cross-surface** — INV-P05 confirm prompt (`--yes` bypasses), INV-P10 rename-source-to-tombstone, then `exec` the target binary.
- **`csq models switch` Codex dispatch** (PR-C7). `csq models switch <slot> <model>` now classifies the slot via `discover_all` and routes Codex slots to a `TomlModelKey` writer that updates `config.toml` rather than the JSON-based ClaudeCode `settings.json`.
- **Desktop Codex UI** (PR-C8). AddAccountModal gains four panels: codex-tos (first-time Terms disclosure), codex-keychain-prompt, codex-running (live device-code + progress), and codex-picker in ChangeModelModal. AccountList carries a surface badge per slot. Five new Tauri commands: `start_codex_login`, `complete_codex_login`, `list_codex_models`, `acknowledge_codex_tos`, `set_codex_slot_model`.

### Two rounds of red-team convergence

The Codex surface shipped under the same zero-tolerance regime as v2.0.1 (`rules/zero-tolerance.md` Rule 5 — no findings above LOW carry across releases). Two redteam rounds plus the post-round-2 dispatcher hardening converged before this release notes doc landed.

#### Round 1 (PR-C9a) — three parallel agents, 15 above-LOW findings, all resolved

Journal 0021 is the findings ledger; journal 0022 is the convergence record. Highlights:

- **CRITICAL — auto_rotate INV-P11 Codex gap.** `find_target` now uses `discover_all` and short-circuits when the current account's surface is not ClaudeCode (v2.1 auto-rotate is ClaudeCode-only by design). `repoint_handle_dir` adds a belt-and-suspenders refusal keyed on Codex-unique symlinks.
- **HIGH — codex device-auth subprocess hardening.** Bounded `BufReader` (64 KiB lines), scrubbed-line parser gated on stdout, `mpsc::sync_channel(4)` with `try_send`, `child.wait()` BEFORE `.join()` on reader threads, `cancel_codex_login` command + AddAccountModal `handleClose` wiring, `complete_codex_login` re-entrancy guard with named error.
- **HIGH — IPC payload audit.** Per-struct **whitelist** via `assert_ipc_keys_whitelisted` helper (replaces the prior blacklist); `whitelist_helper_panics_on_extra_key` regression pins refusal behaviour.
- **HIGH — `emit_to("main")`.** `codex-device-code` event no longer broadcasts to every window.
- **HIGH — INV-P10 signal-window race.** `csq swap` cross-surface path renames the source handle dir to a sweep tombstone (`.sweep-tombstone-swap-<pid>-<nanos>`) instead of `remove_dir_all`, closing the Ctrl-C orphan race and preserving open fds for the running surface process.
- **HIGH — shared cross-module test_env mutex.** New `csq-core/src/platform/test_env.rs` serializes env-var-mutating tests across the workspace.
- **HIGH — R4 raw auth.json scrub on canonical save failure.** New `scrub_and_remove_written` helper called from BOTH the success cleanup path AND the canonical-save error branch.
- **MEDIUM** — `is_device_code_shape` narrowed to exactly `XXXX-XXXX`; `acknowledgeCodexTos` recursion guard via `tosRetry` parameter; `redact_tokens` wrap on outer error format; M4 keychain purge redact; M5 fixed 64 KiB zero buffer for raw-auth-json wipe; M6 500ms `recv_timeout` on `/api/invalidate-cache`; M9 ToS marker I/O log distinguishes `NotFound` from other `io::Error` kinds.

#### Round 1 follow-on — M10 same-surface Codex repoint

Round 1 deferred M10 (same-surface Codex→Codex was routed to `cross_surface_exec`, silently `exec`-replacing the running codex process and dropping the user's conversation) pending an Option A vs Option B human decision. **Option A landed** in the same PR (PR-C9a):

- New `repoint_handle_dir_codex(base_dir, handle_dir, target)` mirrors `repoint_handle_dir` for the spec 07 §7.2.2 Codex symlink set: `.csq-account`, `auth.json`, `config.toml`, `sessions`, `history.jsonl`. Surface guard refuses ClaudeCode handle dirs.
- The `csq swap` dispatcher routes `(Codex, Codex)` to a new `same_surface_codex` helper that delegates to `repoint_handle_dir_codex`. `cross_surface_exec` is now strictly cross-surface.
- UNIX open-after-rename keeps in-flight session fds valid until close; codex-cli re-stats `auth.json` before each API call so the next request authenticates as the new slot.

Journal 0023 records the decision, alternatives, and consequences.

#### Round 2 (PR-C9b) — single-agent, 1 MEDIUM + 3 LOW, all resolved

Journal 0024 is the convergence record. Per `feedback_redteam_efficiency`, round 2 ran a single focused agent on the round-1 residuals plus the M10 code that round 1 had not seen.

- **M-CDX-1 (MEDIUM).** The `codex_links` slice in `repoint_handle_dir_codex` rewrote `.csq-account` BEFORE `auth.json`. A mid-loop rename failure could leave the marker pointing at slot N+1 while the credential still resolved to slot N — silent quota-attribution drift in the daemon plus an F3 mismatch trip on the next swap. Reordered to credential-first (mirrors ClaudeCode's `ACCOUNT_BOUND_ITEMS`); ctime-ordering regression test added.
- **L-CDX-1 (LOW).** Codex surface guard now requires BOTH `auth.json` AND `config.toml` AND each must be a symlink (not a regular file or dir). Catches planted-file bypass + corrupted partial-create handle dirs.
- **L-CDX-3 (LOW).** Extracted dispatcher routing into pure `RouteKind` + `route()` helper. Three matrix unit tests pin ClaudeCode→ClaudeCode, Codex→Codex, and cross-surface routing — any future refactor that re-routes `(Codex, Codex)` through `cross_surface_exec` fails at `cargo test` time before any user can lose a conversation.
- **L-CDX-2 (LOW).** Windows behaviour of `repoint_handle_dir_codex` is documented as a known limitation — symmetric with `repoint_handle_dir` (ClaudeCode), which is also Windows-untested. Both repoint paths will be audited together when the Windows port (M8-03 named-pipe IPC) lands.
- **M8 (deferred from round 1) — decision: ship as-is.** AddAccountModal sets `step = { kind: 'codex-running', ... }` BEFORE invoking the backend, so the UI transitions to the lede + hint paragraphs in the same render frame as the click. The "did it freeze?" ambiguity that the 100ms heartbeat exists to prevent does not exist here. No timer thread added.

### Migration & compatibility

**Quota schema v1 → v2 (one-way migration on the writer side).**

The daemon now writes `schema_version = 2` to `quota.json`. Existing v1 files migrate silently on first daemon tick after upgrade. A v2.0.1 client continues to read v2 files via the dual-read tolerance shipped specifically for this transition. A v2.0.0 client cannot read v2; if you anticipate downgrading below v2.0.1, snapshot your `accounts/quota.json` before upgrading or stay on v2.0.1.

**Daemon requirement for Codex (INV-P02).**

Codex slots require a running csq daemon. `csq run N` for a Codex slot fails with a clear error if the daemon is not healthy. ClaudeCode slots remain daemon-optional in v2.1 (the daemon is required only for centralized refresh + auto-rotate, both of which have always been opt-in for ClaudeCode-only setups). The desktop app starts the daemon on launch as before.

**Codex Terms-of-Service disclosure.**

First-time Codex sign-in via the desktop UI presents a Terms-of-Service screen that the user must acknowledge before the device-auth flow begins. The marker is persisted at `accounts/.codex-tos-acknowledged-<email>` and re-checked on every `start_codex_login` invocation. Acknowledgement is per-account, not global.

**Configuration files.**

- New file: `accounts/credentials/codex-N.json` (one per Codex slot, 0o400 between refresh windows).
- New file: `accounts/codex-wham-raw.json` (raw `wham/usage` body capture, 0o600, redactor-first).
- New file: `accounts/codex-wham-drift.json` (drift detection ledger, 0o600).
- New file: `accounts/.codex-tos-acknowledged-<email>` (per-account ToS acknowledgement marker).
- `quota.json` schema_version flips from 1 to 2 on first write after upgrade.

### Deferred to v2.1.x or v2.2

- **Same-surface Codex swap on Windows (L-CDX-2).** `repoint_handle_dir_codex` is not `#[cfg(unix)]`-gated and the regression tests are all `#[cfg(unix)]`. Windows behaviour is untested. Decision: aligned with `repoint_handle_dir` (ClaudeCode), which is also Windows-untested. Both paths will be audited together with the Windows port workstream (M8-03 named-pipe IPC).
- **PR-VP-C1b — Windows daemon flag flip.** The default-off `windows-daemon` Cargo feature exists in v2.0.1 but has not been flipped to default-on pending fresh-profile Windows 11 VM smoke. v2.1 carries the v2.0.1 Windows behaviour forward (daemon short-circuits on Windows; tray reports "daemon not available"). Codex on Windows therefore depends on this flag flip + the named-pipe Codex transport surface.
- **`RepointStrategy` trait extraction.** The two repoint paths (ClaudeCode + Codex) share lock + pre-flight + rename loop but diverge on the trailing `materialize_handle_settings` + `rebuild_claude_json_for_swap` (ClaudeCode-only). At N=2 surfaces the duplication-with-comments is cheaper than the abstraction; re-evaluate at N=3.
- **IPC whitelist proc-macro.** Round 1's per-struct `assert_ipc_keys_whitelisted` is enforced at unit-test time, not compile-time. A proc-macro would catch additions at derive-time but adds a permanent build-graph dependency that violates `rules/independence.md` Rule 3. Trigger to revisit: a second IPC type slip past the test harness.

## Platform notes

**macOS.** Same as v2.0.1 — the `.app` bundle is ad-hoc signed (no Apple Developer cert provisioned). Right-click → Open is required on first launch on Sequoia / Sonoma, or use `xattr -d com.apple.quarantine "/Applications/Code Session Quota.app"` to drop the Gatekeeper attribute. `.dmg` signing is consistent (codesign --force --deep --sign - + hdiutil create) so the app is launchable without a re-sign step after install.

**Linux.** Same as v2.0.1 — daemon socket at `$XDG_RUNTIME_DIR/csq/csq.sock` (or `~/.claude/accounts/csq.sock` if XDG_RUNTIME_DIR is unset); `SO_PEERCRED` enforces same-UID. CLI binary is statically linked except for libc + system tools. Codex surface tested on x86_64 Linux.

**Windows.** Daemon supervisor still short-circuits per v2.0.1 PR-VP-C1a (the windows-daemon Cargo feature is default-off). Therefore:

- Codex on Windows is **not supported** in v2.1 — Codex slots require a running daemon (INV-P02).
- ClaudeCode on Windows works as in v2.0.1 (no daemon-dependent features: no centralized refresh, no auto-rotate, no usage poller). `csq swap` on ClaudeCode handle dirs works in-flight.
- Cross-surface `csq swap` on Windows is untested and will fail at the `exec` step (`std::os::unix::process::CommandExt` is Unix-only). The new `same_surface_codex` path is also Windows-untested per L-CDX-2.

The full Windows port lands when PR-VP-C1b flips the daemon flag + the named-pipe Codex transport surface lands.

## Upgrading

```
# Existing v2.0.x users (tray)
Click "Update" in the csq tray menu.

# CLI users
curl -L https://github.com/terrene-foundation/csq/releases/download/v2.1.0/csq-<os>-<arch> -o /usr/local/bin/csq
chmod +x /usr/local/bin/csq
```

**No configuration migration required for ClaudeCode-only setups.** Quota.json migrates v1 → v2 silently on first daemon tick. Handle dirs and credentials are untouched.

**Codex sign-in (new in v2.1).** After upgrade, open the desktop AddAccountModal and select the "Codex" provider. The first-time ToS panel will appear; acknowledge to proceed to the device-auth flow. CLI users can run `csq login N --provider codex`.
