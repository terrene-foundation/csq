# 04 — Gemini Surface: Deep Risk Analysis

Phase: /analyze | Agent: deep-analyst | Date: 2026-04-22

Failure modes, race conditions, complexity scoring, and gap matrix for the Gemini integration. Reads against spec 07 §7.2.3/§7.3.4/§7.4 and existing csq source. Gemini is API-key-only; no OAuth, no refresh, no daemon dependency for this surface — but that INTRODUCES a new class of regressions against code that assumes every account has OAuth semantics.

**Codex-specific assumption Gemini invalidates:**

- Codex INV-P02 says "daemon hard prerequisite for refreshable surfaces." Gemini is not refreshable → INV-P02 does NOT apply. Simple boolean inversion — but easy to flip wrong.
- Codex taught the refresher to dispatch by surface. Gemini-introduced dispatch cases: `match surface { Gemini => continue, _ => ... }` in refresher.rs:304 AND in auto_rotate.rs:331.
- Codex's `credentials/codex-<N>.json` mode-flip (INV-P08) has no Gemini analog. `config-<N>/gemini-key.enc` is 0600 always (read once at spawn, not re-read). Must NOT get added to the mode-flip reconciler or it will flip Gemini keys to 0400 and break spawn.

**New load-bearing regression tests:**

- `fn refresher_tick_skips_gemini_surface`: install a Gemini account, assert neither `broker_check` nor `discover_anthropic` processes it.
- `fn usage_poller_anthropic_skips_gemini_source`: same for `daemon/usage_poller/anthropic.rs`.
- `fn gemini_settings_preseed_survives_respawn`: simulate user-edit setting `selectedType: "oauth-personal"` → next `csq run` rewrites back to `"gemini-api-key"`.
- `fn gemini_key_never_fallthrough_to_env_file`: integration test that spawns `gemini` from csq in a `$CWD` containing a `.env` with `GEMINI_API_KEY=BADKEY` and asserts csq's injected key wins.
- `fn effective_model_recorded_on_response`: probe response contains `modelVersion` field; quota.json updates `effective_model`.
- `fn quota_v2_schema_tolerates_gemini_fields`: daemon reads a quota.json that was written by a Gemini-aware desktop; Claude-only daemon code path doesn't crash.
- `fn mode_flip_reconciler_skips_gemini_key_enc`: daemon startup reconciler does not flip `gemini-key.enc` mode.

## 6. ToS Enforcement Surface

Google's ToS bans third-party software from reusing OAuth-subscription access. csq must defensively ensure users cannot route OAuth-subscription traffic through csq-managed gemini EVEN IF they try.

**Attack paths that violate ToS:**

1. User pre-existing `~/.gemini/oauth_creds.json` from standalone gemini-cli use.
2. User edits `config-<N>/.gemini/settings.json` to `"oauth-personal"` (F7).
3. gemini-cli version upgrade changes default auth-type resolution.
4. User's shell exports `GOOGLE_OAUTH_CREDENTIALS_PATH` pointing at oauth_creds.json; child process inherits.
5. csq's `GEMINI_API_KEY` injection fails (undefined Rust behavior, env length limit, platform quirk) → gemini-cli falls back to OAuth.

**Enforcement points (MUST exist in PR-G2/G4):**

- **EP1: Drift detector at every spawn** — Before `exec gemini`, read `config-<N>/.gemini/settings.json`; if `security.auth.selectedType != "gemini-api-key"`, rewrite + log warning. Do NOT trust one-time pre-seed. (ADR-G04, FR-G-CORE-04.)
- **EP2: User-level settings shield** — At csq install OR at first Gemini slot provision, detect `~/.gemini/settings.json` with `selectedType: "oauth-personal"` or `~/.gemini/oauth_creds.json` existence. Prompt user per ADR-G12.
- **EP3: Env sanitation at spawn** — Remove `GOOGLE_APPLICATION_CREDENTIALS` (if provisioned as AI Studio, not Vertex), `GEMINI_OAUTH_TOKEN`, `GEMINI_OAUTH_REFRESH_TOKEN` from child env before injecting `GEMINI_API_KEY`. Preserve `PATH`, `HOME`, `USER`, `LANG`, terminal vars.
- **EP4: Response-body sentinel** — csq-cli wraps gemini's process to parse 429s (see R4), also scans for OAuth-flow telltale strings (`"Opening browser"`, `oauth2.googleapis.com`, `cloudcode-pa.googleapis.com` on an AI-Studio-provisioned slot). If detected, kill child immediately + surface ToS-risk error.
- **EP5: UI never exposes OAuth option** — brief §Key constraint 1 is absolute. AddAccountModal Gemini card MUST NOT render an OAuth option even if the user has prior OAuth creds. No hidden feature-flag path.
- **EP6: Spawn refuses if `oauth_creds.json` present in handle dir chain** — Integration test. Most defensive.
- **EP7: Log + deny on GrowthBook-analog** — if gemini-cli adds A/B test flags that can flip auth type at runtime (no evidence yet, but defense-in-depth given GrowthBook lesson), csq must parse `config-<N>/.gemini/` for any cached flag and refuse to spawn until cleared.

## 8. Silent-Downgrade Risk

Gemini preview models downgrade without error. Same failure class as the documented GrowthBook `tengu_auto_mode_config` that forces Sonnet silently.

**Detection mechanism (MUST exist in PR-G3):**

1. **Per-response capture:** Every `generateContent` response contains `modelVersion` (or equivalent — resolve GG3 first). Parse on each response.
2. **State stored in `quota.json[N]`:** `selected_model` + `effective_model` + `effective_model_first_seen_at` + `mismatch_count_today`.
