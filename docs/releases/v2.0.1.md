## What's new

v2.0.1 is a **safety patch** on v2.0.0. It fixes one CRITICAL credential-handling risk, four HIGH-severity correctness bugs, and nine MED/LOW hardening items. No new features, no user-visible behaviour changes on the happy path, no schema changes on the writer side.

If you are on v2.0.0 or any alpha — update.

### The CRITICAL fix

Auto-rotation could, under a narrow but reachable set of conditions, route an Anthropic OAuth access token through a third-party provider endpoint. Specifically: if a slot had ever been bound to Anthropic OAuth and was later re-bound to a 3P provider (e.g. MiniMax, Z.AI), the stale `credentials/N.json` from the original OAuth binding made the slot a rotation candidate. When auto-rotation selected that slot, it repointed the active terminal's credentials symlink to the slot's `config-N`, whose `settings.json` carried `env.ANTHROPIC_BASE_URL=https://api.minimax.io` or similar — and the next API call sent the Anthropic OAuth bearer token to the 3P host.

v2.0.1 fixes this at two layers: `find_target` now filters rotation candidates to `AccountSource::Anthropic` and also probes each candidate slot's `settings.json` for a non-Anthropic `ANTHROPIC_BASE_URL`, rejecting matches. The active-terminal `tick` path has a belt-and-suspenders check that skips 3P slots outright.

If you have never mixed OAuth and 3P bindings on the same slot number, you were not reachable by this bug.

### Full fix list (journal 0067 — VP-final red-team convergence)

#### Auto-rotate (PR #160)

- **F1 CRITICAL** — 3P slot token exfiltration (above).
- **F2 HIGH** — Cooldown-map key canonicalization. Prevents rotation thrashing when `base_dir` resolves differently between ticks on macOS (`/var/...` vs `/private/var/...`).
- **F3 MED** — Pre-flight target-existence check in `repoint_handle_dir`. Prevents a mid-loop mixed-state handle dir when a target item is concurrently removed.
- **F4 HIGH** — `.swap.lock` flock in `repoint_handle_dir`. Both daemon auto-rotate and `csq swap` go through this path, so the lock serializes concurrent invocations. Fixes the mixed-symlink race where `.credentials.json` points at one slot and `.csq-account` points at another.
- **F8 MED** — New `sweep_races_rotate_on_dead_handle_dir` integration test asserts no panic and correct final state when the sweep and the rotate tick race on the same set of handle dirs.

#### Credential sync (PR #158)

- **H1 HIGH** — `bind_provider_to_slot` and `unbind_provider_from_slot` acquire a file lock on `config-<slot>/settings.json.lock` before the read-modify-write. Prevents silent lost-update under concurrent `csq setkey` / desktop setkey calls on the same slot.
- **H2 HIGH** — `broker::sync::backsync` now distinguishes `CredentialError::NotFound` (canonical absent — proceed to write from live) from `Corrupt` / `Io` (canonical unreadable — abort, refuse to overwrite). Prior behaviour collapsed both to `None` via `.ok()`, silently overwriting a corrupt canonical with a tier-stripped live credential.

#### Quota schema shakedown (PR #157 spec + PR #159 code)

v2.0.1 adds **READ tolerance for the v2 quota.json schema** that v2.1 will write. The schema is frozen in `specs/07-provider-surface-dispatch.md §7.4.1` and carries nested Gemini fields (`CounterState`, `RateLimitState`), an `effective_model_first_seen_at` string, and an `extras: Option<serde_json::Value>` escape hatch for surface-specific payload fragments. v2.0.1 continues to **write v1** — the write-path flip lands in v2.1.

- **R1 HIGH** — Gemini schema shape reconciled between spec 05 §5.8 and spec 07 §7.4.1 (nested structs, not flat scalars).
- **R2 HIGH** — `extras` escape hatch added; future surfaces (notably Codex `wham/usage`) can stash unmigrated fields without forcing a schema v3 bump.
- **R3 HIGH** — `schema_version > 2` now degrades to an empty quota state with a WARN, instead of hard-failing `csq statusline` etc. Rollback-friendly.
- **R4 MED** — `AccountQuota::new_at(updated_at)` constructor prevents the silent-zero-epoch regression that `..Default::default()` would have introduced.
- **R5 MED** — Non-numeric account keys in quota.json are rejected with an actionable error message.
- **R6 MED** — Write-path explicitly forces `schema_version = 1` at the serialization boundary; the in-memory struct may carry v2 after a v2-file load, but v2.0.1 never writes it.
- **L1 LOW** — Round 2 tightened R5 to reject keys outside `AccountNum`'s 1..=999 range (the first round accepted "0" and "1000" as parseable u16s).

#### Pre-existing guard fixes (already in v2.0.0 stable ship, confirmed intact)

- **PR-B5** — `broker::sync::backsync` preserves `subscription_type` and `rate_limit_tier` when live carries `None` (the post-re-login state).
- **PR-B6** — `bind_provider_to_slot` reads existing settings before overlaying the provider binding (RMW, not blind-write).

#### Structural rotation fix (PR-A1)

- **P0-1** — Auto-rotation is now handle-dir-native. The v2.0.0 gate-off guard ("refuse to run when any `term-*/` exists") is removed; the rotator walks `accounts/term-*` and calls `handle_dir::repoint_handle_dir` to swap symlinks atomically, without ever writing to `config-<N>/`. Journal 0064 Option A.

## Upgrading

```
# Existing v2.0.0 users (tray)
Click "Update" in the csq tray menu.

# CLI users
curl -L https://github.com/terrene-foundation/csq/releases/download/v2.0.1/csq-<os>-<arch> -o /usr/local/bin/csq
chmod +x /usr/local/bin/csq
```

No configuration migration required. quota.json, credentials, and handle dirs are compatible both ways between v2.0.0 and v2.0.1.
