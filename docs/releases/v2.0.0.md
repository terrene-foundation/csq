# csq v2.0.0

**Release date:** 2026-04-22
**Prior release:** v2.0.0-alpha.21 (2026-04-20)
**Checksum / signature:** see artifact `SHA256SUMS` and `.sig` files on the GitHub Releases page.

---

## What's new

csq v2.0.0 is the first stable release of the Rust rewrite. The legacy bash + Python v1.x is retired. Everything the v1.x ran with — rotation engine, token refresher, dashboard, statusline hook, OAuth flow — is now one signed binary + one desktop app.

### Headline features

- **Single-binary CLI.** `csq run N`, `csq swap N`, `csq status`, `csq login`, `csq setkey`, `csq install`, `csq doctor`, `csq update`, `csq statusline`. No Python runtime required.
- **Desktop tray app** on macOS, Linux, and Windows. System-tray icon + menu, quota dashboard, "Add Account" flow via browser OAuth, "Change model" affordance for Ollama slots, in-flight account swap, recurring update check.
- **Central token refresher.** Refreshes every OAuth account's tokens before they expire; eliminates the LOGIN-NEEDED stampede that plagued v1.x.
- **Handle-dir session model.** Each `claude` session gets an ephemeral `term-<pid>/` handle dir; `csq swap` repoints symlinks atomically instead of copying credentials. Spec 02.
- **Third-party providers.** MiniMax and Z.AI via `csq setkey`, Ollama via `csq bind ollama`. Quota polling against each provider's upstream.
- **In-app update detection.** Tray "Check for updates" + recurring background check with exponential backoff. Detects new releases and prompts the user. (One-click in-app install is deferred until the Foundation's release-signing key is provisioned — see L2 below.)
- **Statusline integration.** CC's statusline now shows live csq quota and rotation state via `csq statusline`; legacy per-slot shell-wrapper configurations are automatically migrated on upgrade (journal 0059/0060).
- **Signed macOS builds** (ad-hoc signature — see L1 for the first-launch bypass).

### Stability gates held for 2.0.0

- 675+ Rust tests across csq-core, csq-cli, csq-desktop
- 93+ Svelte tests
- Zero clippy warnings, zero fmt drift
- All known red-team findings above LOW severity resolved in-source. Residual LOW findings tracked for 2.0.1 per `workspaces/csq-v2/journal/0063`.

## Installing

### macOS

```
curl -L -o csq.dmg https://github.com/terrene-foundation/csq/releases/download/v2.0.0/csq-desktop-macos.dmg
open csq.dmg
```

Drag **Code Session Quota.app** to Applications. On first launch, right-click → **Open** → confirm (see L1). Subsequent launches require no bypass.

### Linux

```
curl -L -o csq.AppImage https://github.com/terrene-foundation/csq/releases/download/v2.0.0/csq-desktop-linux.AppImage
chmod +x csq.AppImage
./csq.AppImage
```

### Windows

Download `csq-desktop-windows-setup.exe` and run it. SmartScreen will show a warning on first launch — click **More info → Run anyway** (see L3).

### CLI only

```
curl -L https://github.com/terrene-foundation/csq/releases/download/v2.0.0/csq-<os>-<arch> -o /usr/local/bin/csq
chmod +x /usr/local/bin/csq
csq --version   # -> csq 2.0.0
csq install     # seeds ~/.claude/accounts, migrates legacy statusline
csq login 1     # OAuth in your default browser
csq run 1       # launches claude under account 1
```

## Known limitations

These are explicit non-claims for v2.0.0. Each has a manual workaround that works today.

### L1 — Apple notarization is not yet present

csq 2.0.0 is ad-hoc signed on macOS. On first launch, right-click the app and choose "Open" (then confirm) to bypass the "unidentified developer" warning. Subsequent launches require no bypass. Apple Developer ID signing and notarization are pending provision of the Foundation's developer certificate.

### L2 — `csq update install` one-click upgrade is deferred

In csq 2.0.0, the in-app "Update" button and `csq update check` CLI detect new releases and prompt you. The one-click `csq update install` path is gated behind the Foundation's production release-signing key; until that key is provisioned, install 2.0.1+ manually by downloading the new DMG / AppImage / MSI from GitHub Releases. Update check and detection work normally. Credentials and config persist across reinstalls — nothing is lost.

### L3 — Windows SmartScreen warning on first install

The Windows installer is not yet EV-code-signed. On first run, Windows SmartScreen will display a warning — click **More info → Run anyway** to proceed. EV code signing is pending provision of the Foundation's Windows signing certificate.

### L4 — Windows end-to-end path has limited manual smoke validation

csq 2.0.0 ships with Windows support that has been code-tested (Rust unit + integration tests green on the Windows CI matrix) but has had limited manual validation on fresh Windows 11 profiles. As a stability measure for 2.0.0, the desktop supervisor will NOT claim daemon ownership on Windows — tokens will not auto-refresh from the desktop app. Run `csq daemon start` from a terminal for token refresh on Windows, or run csq's CLI as-needed (login / run will refresh on demand). The full Windows named-pipe daemon wiring ships in a post-2.0.0 release. File issues at https://github.com/terrene-foundation/csq/issues with the `windows` label if you hit anything.

### L5 — System tray on Linux requires AppIndicator

On Linux, the tray icon relies on the AppIndicator protocol. On GNOME Wayland, install the "AppIndicator and KStatusNotifierItem Support" extension if the tray icon does not appear. KDE Plasma, Cinnamon, and X11 desktops work without additional setup.

### L6 — Auto-rotation is deferred in handle-dir mode

csq 2.0.0 uses the handle-dir session model (spec 02) — each terminal gets its own `term-<pid>` handle dir with symlinks to the active account's credentials. The legacy auto-rotation algorithm (from pre-handle-dir csq) rotates config directories by copying credentials, which under handle-dir semantics would corrupt account identity. The daemon explicitly refuses to run auto-rotation when handle dirs are present (journal 0064), logs a WARN, and leaves your credentials untouched. A handle-dir-native rotator ships in 2.0.1. For 2.0.0, use `csq swap N` manually when you want to switch accounts.

## Upgrading from alpha.x

Previous alpha installs upgrade in place — your accounts, credentials, quota history, and profile data carry over.

If you were running alpha.18 or earlier, `csq install` on 2.0.0 will detect stale per-slot `statusLine` commands pointing at the v1.x shell wrapper (`statusline-quota.sh`) and rewrite them to `csq statusline`. This is a one-shot migration (journal 0059/0060). Running `csq install` on a clean alpha.21+ install is a no-op.
