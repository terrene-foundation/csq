# csq v2.0.1 — Safety Patch Plan

## Guiding principles

1. **Safety patch, not feature release.** No behaviour-breaking changes. Quota.json schema stays v1 in writes; v2.0.1 adds v2 READ tolerance only.

2. **Shakedown for v2.1.** PR-B8 dual-read deserialiser lands in v2.0.1 so v2.1's quota.json v1→v2 write-path flip has a full release of production shakedown on the read side before any user file is converted.

3. **Stub-ahead for shared-spine PRs.** PR-A1 rewrites auto_rotate; Codex PR-C1 (v2.1) later adds a same-surface filter to the same functions. Rather than reshaping twice, PR-A1 lands with a `Surface::ClaudeCode`-only filter stub (one-line TODO citing PR-C1). PR-C1 flips the stub.

4. **Fix, don't defer.** Per `zero-tolerance.md` Rule 5, no "documented and accepted" residuals. Deferred items are either rolled into v2.1 explicitly (see journal 0068) or external-infra gated.

5. **Redteam convergence is a cut-criterion.** Per H12, PR-VP-final is mandatory before release tag.
