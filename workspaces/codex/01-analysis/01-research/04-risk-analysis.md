## 1. Failure modes

Exhaustive enumeration of every way a Codex slot can degrade, with user-visible symptom.

## 2. Concurrency / Race Matrix

| Race | Interleaving                                                                         | Outcome                                                                                                                                                                                                                                                                                                                                                                       | Status                                                                                                                                                                                                                                                                                                                                |
| ---- | ------------------------------------------------------------------------------------ | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| R1   | **Daemon refresh (T1) × `csq swap` on same account (T2)**                            | T1 acquires per-account `tokio::Mutex`, writes new token. T2 cross-surface swap does `exec <new-binary>` — fine. Same-surface swap doesn't write credentials, only repoints. **SAFE**                                                                                                                                                                                         | OK if mutex honored everywhere.                                                                                                                                                                                                                                                                                                       |
| R2   | **Two `csq run <same codex slot>` in parallel**                                      | Each gets a `term-<pid>/`. Both symlink `auth.json → credentials/codex-<N>.json`. Both may trigger codex's in-process refresh on startup. **BOTH CC PROCESSES WILL ATTEMPT REFRESH** → openai/codex#10332 single-use RT race                                                                                                                                                  | **UNMITIGATED.** INV-P02 says daemon is sole refresher, but two codex processes preemptively refreshing 30 min before expiry is exactly the failure mode. Spec is silent on codex's OWN in-process refresh behavior — does `cli_auth_credentials_store = "file"` disable in-process refresh? No source citation. See ADR-C15 blocker. |
| R3   | **Daemon sweep × codex mid-write to `sessions/<id>.jsonl`**                          | `sessions` is a symlink OUT of handle dir. `rename(term-<pid>, tombstone)` on symlink is fine. `remove_dir_all(tombstone)`: Rust 1.74+ uses `openat` + `unlinkat AT_REMOVEDIR`; rejects symlinks. csq-core/src/session/handle_dir.rs:697, 742 relies on this. **MUST write integration test: sweep of codex handle dir does NOT delete persistent codex-sessions/ contents.** | Must-verify-with-test                                                                                                                                                                                                                                                                                                                 |
| R4   | **User hand-edits `config-<N>/config.toml` × daemon wants to rotate `model` key**    | TOML parse-and-rewrite is not atomic w.r.t. editor's `mv`. User saves → daemon reads mid-write → garbled TOML → codex spawn fails next time.                                                                                                                                                                                                                                  | **UNMITIGATED.** Spec 07 §7.3.3 writes config.toml at login but says nothing about subsequent daemon rewrites. INV-P06 says daemon writes `model = "..."` — needs file lock or watchdog.                                                                                                                                              |
| R5   | **Cross-surface `csq swap` during active codex streaming response**                  | INV-P05 says `exec <new-binary>` in place. But codex is mid-SSE stream → kernel closes FD → stream aborts with EPIPE. User sees truncated response, no resume.                                                                                                                                                                                                                | Warning fires BEFORE confirmation; if user confirms while streaming, data loss happens silently for that turn. Add detection or louder warning.                                                                                                                                                                                       |
| R6   | **Daemon restart × terminal doing in-session refresh**                               | Daemon down → codex in-process refresh (if enabled) succeeds → writes to `credentials/codex-<N>.json` via symlink → next daemon startup reads new RT but cooldown state is lost                                                                                                                                                                                               | In-memory cooldowns drop on daemon restart (existing Anthropic bug, carries over).                                                                                                                                                                                                                                                    |
| R7   | **`credentials/codex-<N>.json` mode flip (0400 ↔ 0600) race**                        | Daemon flips to 0400 outside refresh window. Another `csq login N --provider codex` needs to write → gets EACCES.                                                                                                                                                                                                                                                             | 0400 outside refresh windows breaks `csq login` that wants to write. Login path must flip 0600 → write → flip 0400 atomically under same mutex. **Resolved by ADR-C13.**                                                                                                                                                              |
| R8   | **Sibling terminal reads `auth.json` via symlink while daemon is mid-atomic-rename** | `atomic_replace` uses `rename(tmp, target)` — atomic on same filesystem. Symlink resolution at every `open()`. Reader always sees either old or new complete file. **SAFE**                                                                                                                                                                                                   | OK                                                                                                                                                                                                                                                                                                                                    |
| R9   | **Two daemons (different UIDs on same host)**                                        | Each has its own socket under per-user dir; no conflict. Both poll `wham/usage` for their respective subscriptions — no problem.                                                                                                                                                                                                                                              | Only a problem if users share a subscription (not in scope).                                                                                                                                                                                                                                                                          |

## 6. Pre-Existing Bugs That Codex Will Expose

- **csq-core/src/daemon/refresher.rs:283** `discover_anthropic(base_dir)` — filters to Anthropic only. After PR1 refactor, this comment and the call name will lie if the filter doesn't explicitly include Codex. PR1 MUST rename to `discover_refreshable` or similar, keyed by `Surface`.
- **csq-core/src/daemon/refresher.rs:304-307** `if info.source != AccountSource::Anthropic || !info.has_credentials { continue; }` — the exact Anthropic-only guard. Codex needs its own refresh loop OR this filter becomes surface-dispatched with a Codex-specific refresh call.
- **csq-core/src/broker/check.rs:278** `is_rate_limited` — substring match. Journal 0052 lesson (invalid_scope misclassified) WILL repeat with OpenAI-specific error codes. `{"error":{"type":"rate_limit_exceeded"}}` vs `{"error":{"code":"insufficient_quota"}}` vs `403 banned` — all three will be misbucketed. **Pre-existing classifier brittleness extends to Codex.**
- **csq-core/src/rotation/swap.rs:64** `creds.claude_ai_oauth.subscription_type` — hardcoded struct field. Codex creds have no `claude_ai_oauth` key. `swap_to` MUST refuse on a Codex slot OR dispatch by surface.
- **csq-core/src/credentials/file.rs:114** `live_path` returns `config-<N>/.credentials.json` — hardcoded. Codex needs `config-<N>/codex-auth.json`. Parameterize by `Surface`.
- **csq-core/src/credentials/mod.rs:27** `pub claude_ai_oauth: OAuthPayload` — struct name. Either rename to `tokens: OAuthPayload` (breaking change to `credentials/N.json` serialization → migration) or fork into `CredentialFile::Anthropic { claude_ai_oauth }` / `CredentialFile::Codex { tokens }`.
- **csq-core/src/credentials/file.rs:135** `rate_limit_tier: Some("default_claude_max_20x".into())` — test fixture default. Subscription-preservation guard in refresh.rs:202 and swap.rs:64 hardcodes reading `subscription_type` / `rate_limit_tier`. Codex has `plus`/`pro`/`business`/`enterprise`. Guard becomes surface-dispatched or Codex silently loses tier info.
- **csq-core/src/session/handle_dir.rs:37-41** `ACCOUNT_BOUND_ITEMS = [".credentials.json", ".csq-account", ".current-account", ".quota-cursor"]` — hardcoded. Codex needs `auth.json`, `config.toml`, `sessions`, `history.jsonl`. Parameterize by `Surface`.
- **csq-core/src/session/handle_dir.rs:697, 742** `remove_dir_all` — see R3. **CRITICAL verification needed:** confirm by unit test that `remove_dir_all` on a handle dir containing `sessions` as a symlink to `config-<N>/codex-sessions/` does NOT traverse the symlink and delete persistent transcripts.
- **csq-core/src/daemon/usage_poller/anthropic.rs:21** module-level constants assume Anthropic. Sibling codex module is fine — but cooldowns HashMap keyed by `u16` account ID means slot-9 Anthropic and slot-9 Codex would share cooldown. Keyed map must become `(Surface, AccountNum)`.
- **csq-core/src/accounts/discovery.rs:88** `if lower.contains("api.anthropic.com")` — provider classification by URL. Codex doesn't fit this model. Discovery layer adds first-class `Surface` dimension.
- **csq-core/src/daemon/auto_rotate.rs:331** `discover_anthropic` — auto-rotation only on Anthropic. Cross-surface rotation policy: **G13.**
- **csq-core/src/error.rs:260** `error_kind_tag` — closed enum of Anthropic-flavored tags. Codex OAuth errors won't map. Extend enum; add round-trip test.

## Summary — Top Actions Before PR1 Lands

1. **G1 verification first (ADR-C15)** — read openai/codex source for `cli_auth_credentials_store = "file"` semantics re: in-process refresh. If codex still refreshes in-process, spec 07 INV-P01/P02 needs a different strategy.

3. **Pin `remove_dir_all` symlink-safety** with an integration test before PR2 ships.
4. **Resolve the 5 MUST-RESOLVE-IN-PR items** from §5 (graceful-degradation logging, cold-cache model list, refuse-on-keychain-decline via ADR-C11 update, 0400/0600 mutex dance via ADR-C13, keychain-purge refusal path).
5. **Enumerate OpenAI error-response shapes** for the 4 buckets (`rate_limited`, `invalid_grant`, `suspended`, `schema_drift`) into `error_kind_tag` before PR3 lands — or Codex will reproduce journal 0052 verbatim.
